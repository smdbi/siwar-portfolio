Overview

We piloted process mining on a P2P/O2C flow in an ERP WMS SaaS. The goal was to expose bottlenecks, cut defect intake, and forecast delivery windows using DORA metrics and Monte Carlo simulations.

High-level overview of the process mining flow, from data sources to analysis and forecasting.

Problem

Cycle time volatility and a growing defect backlog reduced predictability. We needed a data-backed view of where time leaked, which variants drove rework, and realistic delivery windows stakeholders could trust.

Cycle-time spread P90–P50 = 8.4d last 2 sprints
Rework variants 27% from conformance
Backlog growth +18% rolling 4 weeks

O1 — Predictable delivery

  • KR1: Shrink cycle time by 25% (median) with same scope.
  • KR2: Publish P80/P95 UAT windows each sprint.

O2 — Fewer defects, faster triage

  • KR1: Reduce open defects by 30% via conformance + hot-component focus.
  • KR2: 95% of tickets within SLA.
Constraints
Event-log completeness (Case ID · Activity · Timestamp), multi-system joins, release cutoffs.
Assumptions
Stable team capacity, constant WIP limits, comparable scope mix across sprints.

Approach

  1. BPMN as-is / to-be

    Mapped P2P & O2C; added acceptance criteria and RTM linkages so gaps and risk became explicit.

    • Entry/exit oracles for each flow
    • Traceability across BRD → stories → tests

  2. Process mining

    Imported event logs; ran variant frequency & conformance checks to locate rework and slow paths.

    • Case ID · Activity · Timestamp (+ Resource)
    • Variant drift alerts on weekly cadence

  3. Defect analytics

    Lightweight Python pipeline for triage: hot components, age risk, and dupe hints for backlog hygiene.

    • Age bands & “hot” components
    • NLP-lite duplicate detection

  4. Delivery forecasting

    Throughput-based Monte Carlo with DORA context to publish P80/P95 UAT windows and assumptions.

    • Weekly refresh; visible assumptions
    • Stakeholder comms via release notes

Risks & Mitigations

  • Data quality gaps → extraction checklist & sampling
  • Scope churn → change control & impact notes

Operating Cadence

  • Conformance + KPI snapshot every Friday
  • Forecast update + deltas every second sprint
